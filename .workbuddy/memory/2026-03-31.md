### 部署问题修复
**问题1**：Railway部署失败  
**原因**：`routes/materials.js` 第20行上传路径指向了不存在的 `uploads` 目录（正确应为 `data/uploads`）  
**修复**：已修正路径配置，提交 commit c9f9a4b

**问题2**：Route.get() requires a callback function but got a [object Undefined]  
**原因**：`routes/materials.js` 导入 `requireAuth`，但 `middleware.js` 导出的是 `requireLogin`  
**修复**：统一使用 `requireLogin`，提交 commit e1f605a

### v2.1 功能升级（commit 27400f0）
**新增功能**：
1. **审批流程完善**：用户申领后状态为 pending，管理员可通过/驳回
2. **申领管理模块**：管理员专用页面，可查看/筛选/编辑/导出所有申领数据
3. **广播公告功能**：物资申领页面顶部显示广播，管理员可编辑公告内容
4. **手动添加物资**：管理员可直接手动添加物资，无需上传Excel
5. **物资管理**：管理员可编辑/删除已添加的物资

**API新增**：
- `GET/POST /api/materials/broadcast` - 广播管理
- `POST /api/materials/material` - 手动添加物资
- `PUT/DELETE /api/materials/material/:id` - 编辑/删除物资

**前端新增**：
- 申领管理导航菜单（仅管理员可见）
- 广播公告卡片（支持多行文本）
- 物资管理弹窗（添加/编辑/删除）
- 申领记录导出CSV功能

### 使用流程
1. 管理员先创建部门并设置额度
2. 管理员上传物资清单 Excel
3. 管理员分配用户到部门
4. 普通用户登录后申领物资
5. 管理员审批申领

## v2.2 功能升级

**时间**：2026-03-31 11:30

### 新增功能：管理员重置用户密码
- **后端API**：`PUT /api/users/:id/password` - 管理员重置指定用户密码
- **前端界面**：用户管理页面增加"重置密码"按钮，点击后弹出密码重置对话框
- **安全限制**：管理员不能通过此接口重置自己的密码（需使用"修改密码"功能）
- **操作日志**：重置密码操作会被记录到操作日志中
