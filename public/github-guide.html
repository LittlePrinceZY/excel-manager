📦 GitHub 推送代码教程

小白也能看懂的图文步骤，把代码上传到 GitHub

1
在 GitHub 创建仓库
先给代码找个"家"
github.com/new
Public 公开（免费）
Private 私有
Add a README file ❌ 不要勾选
Add .gitignore ❌ 不要勾选
Choose a license ❌ 不要勾选
💡 注意

下面的三个复选框都不要勾选！因为我们已经在本地创建了 Git 仓库。

⬇️
2
复制 GitHub 提供的命令
创建后会看到这个页面
github.com/你的用户名/excel-manager

…or push an existing repository from the command line

git remote add origin https://github.com/你的用户名/excel-manager.git
git branch -M main
git push -u origin main
✅ 你不需要自己输入这些命令

GitHub 会自动显示，你只需要点击复制按钮，然后粘贴到 PowerShell 运行。

⬇️
3
在本地运行命令
把代码推送到 GitHub

打开 PowerShell，依次运行以下命令：

# 进入项目目录
PS> cd "c:\Users\Administrator\WorkBuddy\20260331083046"

# 添加远程仓库（把"你的用户名"换成你的GitHub用户名）
PS> git remote add origin https://github.com/你的用户名/excel-manager.git

# 切换到 main 分支
PS> git branch -M main

# 推送代码
PS> git push -u origin main
⚠️ 重要：替换用户名

把命令中的 你的用户名 换成你的 GitHub 用户名，比如 zhangsan

⬇️
4
输入 GitHub 凭据
第一次推送需要验证身份

运行 git push 后，会提示输入：

Username for 'https://github.com': 你的GitHub用户名
Password for 'https://github.com': 你的Personal Access Token
🔐 注意：密码不是登录密码！

GitHub 已经不支持用登录密码推送代码了，你需要创建一个 Personal Access Token（个人访问令牌）。

👇 点击下面的标签查看如何获取 Token：

图文步骤
视频演示
github.com/settings/tokens
  1. 点击右上角头像 → Settings
  2. 左侧最下方 → Developer settings
  3. 左侧 → Personal access tokens → Tokens (classic)
  4. 点击 Generate new token (classic)
  5. 填写 Note：excel-manager-deploy
  6. 勾选 repo（完整仓库访问权限）
  7. 点击 Generate token
  8. 立即复制生成的 token！（只显示一次）

🎬 视频演示区域

建议搜索："GitHub Personal Access Token 创建教程"

⬇️
5
验证推送成功
看到绿色对勾就成功了！

如果看到类似下面的输出，说明成功了：

Enumerating objects: 21, done.
Counting objects: 100% (21/21), done.
Writing objects: 100% (21/21), 45.23 KiB | 15.08 MiB/s, done.
To https://github.com/你的用户名/excel-manager.git
* [new branch] main -> main
branch 'main' set up to track 'origin/main'.
🎉 恭喜！代码已上传到 GitHub

刷新 GitHub 页面，应该能看到你的代码文件列表了。接下来就可以去 Railway 部署了！

常见问题
遇到问题先看这里

Q: 提示 "remote origin already exists"

A: 运行 git remote remove origin 删除旧的，再重新添加

Q: 提示 "Authentication failed"

A: 确认使用的是 Personal Access Token，不是登录密码

Q: 提示 "Permission denied"

A: 创建 Token 时要勾选 repo 权限
